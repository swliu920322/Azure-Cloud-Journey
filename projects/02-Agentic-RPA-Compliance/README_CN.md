# 项目 02: RAG 驱动的智能体 RPA 合规网关
**所属仓库: `Azure-Cloud-Journey`**

## 1. 系统概述
本项目实现了一个企业级、高并发的反洗钱 (AML) 拦截网关。它解决了传统基于规则的 RPA 存在的“语义盲区”与 UI 界面脆弱性问题。通过将认知决策层与物理执行层彻底解耦，系统能够在 API 入口处直接拦截复杂的“蓝精灵 (Smurfing)”式结构化欺诈。

## 2. 核心架构 (非对称漏斗)
系统强制数据流经四个高度隔离的物理层级：

1. **入口级脱敏 (隐私保护设计):** 我们在 API 网关处执行内嵌的 C# 脚本进行动态 MD5 散列。系统在将数据载荷路由至 AI 引擎前，剥离所有个人身份信息 (PII)。
2. **非对称 ML 引擎 (iForest + SVM):** 部署隔离森林 (iForest) 处理粗粒度异常嗅探，瞬间清空 95% 的常规流量。系统将极其耗费算力的支持向量机 (SVM) 高维切割操作，严格限制在被标记的 5% 异常数据上。此架构避免了在正常流量上无谓的算力燃烧。
3. **认知合规溯源 (RAG):** 在金融合规领域，黑盒概率得分不具备法律效力。针对高风险异常，系统触发检索增强生成 (RAG) 模块。它通过检索现行 AML 法规的向量数据库，将数学偏差翻译为具备法律约束力、人类可读的审计追踪。
4. **API 优先执行:** UiPath 机器人彻底放弃 UI 界面抓取。智能体大脑触发异步的 `HTTP POST /api/lockBusiness` 请求，指令核心网关在毫秒级物理冻结高危资产。

## 3. 性能压测指标
我们使用 10,000 条模拟交易数据（混合常规流量与伪装洗钱流量）对流水线进行了压力测试。
* **人工合规审查基线:** 约 80,000 秒 (22.2 小时)
* **智能体 RPA 延迟:** 515 秒 (8.5 分钟)
* **工程收益:** 吞吐量飙升 **155 倍**。系统彻底消除了人类操作员的认知疲劳，并在流量激增时保持绝对确定性的拦截率。

## 4. Azure 云架构映射 (对接 AZ-305 认证)
虽然原型采用本地 ML 模型与 UiPath，但该架构可原生映射至 Azure 企业级组件，以实现云端弹性部署：
* **安全 API 网关** -> *Azure API Management (APIM)*
* **脱敏与路由网关** -> *Azure Functions (C# 无服务器计算)*
* **混合 ML 引擎** -> *Azure Machine Learning Workspace (托管端点)*
* **RAG 向量数据库** -> *Azure AI Search + Azure OpenAI Service*
* **执行编排层** -> *Azure Logic Apps (API 优先编排)*
